1. Get the branch with the seed tooling
git fetch origin claude/ecstatic-davinci-Oqcbd
git checkout claude/ecstatic-davinci-Oqcbd
npm install          # ensures @libsql/client + tsx are present
2. Make sure .env points at Turso
The script reads TURSO_DATABASE_URL and TURSO_AUTH_TOKEN from .env (same ones already in your file):

TURSO_DATABASE_URL="libsql://repohive-heaventree.aws-eu-west-1.turso.io"
TURSO_AUTH_TOKEN="eyJhb..."
3. Find your tenant id (for restoring into your account)
Your tenant id is your Clerk user id (user_…), or your org id (org_…) if you sign in with an org active. Two ways to get it:

Clerk dashboard → Users → your user → copy the User ID, or
Start the app, sign in, open the dashboard, and watch the server log — it prints Fetching repos for <tenantId>.
4. Run the seed
Point --from at your reposcout.db (carries embeddings) and --copy-to at your tenant id:

npx tsx scripts/seed-library.ts --from /path/to/reposcout.db --copy-to user_xxxxxxxxxxxx
You'll see something like:

Loaded 583 repos from DB /path/to/reposcout.db.
__library__: +583 new (583 with embeddings; was 0, now 583).
tenant user_xxx: +583 copied (now 583 total).
Done.
It's idempotent — safe to re-run; existing rows are skipped (+0).

Variations:

Seed only the shared library (no personal copy): drop --copy-to.
No reposcout.db handy (e.g. running in the Render shell): drop --from — it uses the committed 583-repo seed, but embeddings come in empty, so afterward trigger the in-app embedding sweep (POST /api/embed/sweep) to rebuild them.
5. Restart the app
The server loads the embedding cache at startup, so after seeding, restart / redeploy so semantic search picks up the 583 embeddings.

6. Verify
Sign in → your library should show ~583 repos with the App Killer badges. Or query directly:

turso db shell repohive-heaventree "SELECT source, COUNT(*) FROM repos GROUP BY source"
One caution: this writes 583 rows to your live production DB. It's additive and idempotent (never deletes or overwrites), but if you'd like, do a trial run against a throwaway local file first to see the output:

TURSO_DATABASE_URL=file:/tmp/trial.db TURSO_AUTH_TOKEN= npx tsx scripts/seed-library.ts --from /path/to/reposcout.db --copy-to user_xxx

What I'd suggest

So: use the Turso MCP server for verification and ad-hoc queries from Desktop, and the seed script (local terminal or Render shell) for the actual bulk load.
